# 简短思路.md

## 53. 最大子数组和

---

## 1. 题目核心

给定一个整数数组 `nums`，要求找出一个**和最大的连续子数组**，返回这个最大和。

- 题型：**数组 + 连续子数组 + 动态规划**
- 关键限制：**必须连续**
- 关键边界：**子数组至少包含一个元素**
- 本质：在所有连续子数组里，找和最大的那一段

---

## 2. 核心知识点

- **动态规划**
- **状态转移**
- **连续子数组**
- **Kadane 算法**
- **空间优化**

这题真正关键的不是把所有子数组都枚举出来，而是：

**思考“以当前位置结尾”的最大子数组和怎么得到。**

---

## 3. 答题技巧

### 看到这些信号，要想到动态规划

| 题目信号 | 应想到的方法 |
|---|---|
| 连续子数组 | DP / 区间状态 |
| 求最大和 | 考虑前面那段要不要接 |
| 前面负数会拖后腿 | 不如从当前位置重新开始 |
| 只和前一个状态有关 | 可以空间优化 |

### 判断依据

如果题目是：

- 数组中的**连续区间**
- 要求某种**最值**
- 当前答案和“前一位置的最优状态”有关

那就要想到：

**定义 `dp[i]` 或 `cur`，表示“以 `i` 结尾”的最优解。**

---

## 4. 最优思路

定义：

```cpp
dp[i] = 以 nums[i] 结尾的最大子数组和
```

那么当前位置只有两种选择：

1. **自己单独开始一段**
   ```cpp
   nums[i]
   ```

2. **接在前面的最优连续段后面**
   ```cpp
   dp[i - 1] + nums[i]
   ```

所以状态转移是：

```cpp
dp[i] = max(nums[i], dp[i - 1] + nums[i]);
```

因为 `dp[i]` 只依赖 `dp[i-1]`，所以可以把 `dp` 数组压缩成一个变量：

```cpp
cur = max(nums[i], cur + nums[i]);
```

同时用 `ans` 记录全局最大值。

### 为什么这样成立

因为“以 `i` 结尾”的连续子数组，前面那一段如果存在，就一定是“以 `i-1` 结尾”的连续子数组。

所以当前最优解只可能来自：

- 从自己开始
- 接上前面最优段

没有第三种情况。

---

## 5. 我遇到的疑问整理

### 1）为什么这题不是滑动窗口？

因为这题没有明确的“合法 / 不合法”窗口条件。

滑动窗口常见于：

- 覆盖
- 计数
- 最短长度
- 不同字符个数限制

这题问的是：

**哪一段连续子数组的和最大。**

所以更适合用 DP，而不是窗口维护。

---

### 2）`cur` 到底是什么？

`cur` 不是全局最大值，它表示：

**以当前位置结尾的最大子数组和**

例如走到 `nums[i]` 时，`cur` 只管“最后一个数必须是 `nums[i]`”的情况。

---

### 3）为什么状态转移是

```cpp
cur = max(nums[i], cur + nums[i]);
```

因为当前位置只有两种可能：

- 自己重新开始
- 接着前面那段继续走

如果前面的和是负贡献，就不要它；如果前面的和是正贡献，就接上。

---

### 4）为什么前面的和是负数时要断开？

因为负数会拖后腿。

例如：

```cpp
cur = -3
nums[i] = 5
```

那：

- 接上前面：`-3 + 5 = 2`
- 自己开始：`5`

显然自己开始更好，所以前面那段应该扔掉。

---

### 5）为什么初始化不能写成 0？

因为题目要求子数组至少包含一个元素。

如果数组全是负数：

```text
[-3,-2,-5]
```

正确答案是 `-2`，不是 `0`。

所以初始化要写成：

```cpp
int cur = nums[0];
int ans = nums[0];
```

---

### 6）为什么循环必须从 `i = 1` 开始？

因为第 0 个元素已经拿来初始化了：

```cpp
int cur = nums[0];
int ans = nums[0];
```

这表示第 0 个位置的状态已经处理过了。

如果还从 `i = 0` 开始，就会把第一个元素重复计算一次。

例如数组 `[1]`：

- 初始化后 `cur = 1`
- 再从 `i = 0` 算一次，会变成 `max(1, 1+1)=2`
- 明显错了

---

### 7）`cur` 和 `ans` 有什么区别？

- `cur`：**必须以当前位置结尾** 的最大子数组和
- `ans`：到目前为止的 **全局最大** 子数组和

这两个不是一回事。

---

## 6. 易错点

### 1）把题目理解成“可以随便挑几个大数相加”
错因：**思路错误**

这题必须是**连续子数组**，不能跳着选。

---

### 2）初始化成 0
错因：**边界错误**

全负数组时会出错。

---

### 3）`cur` 和 `ans` 混了
错因：**逻辑前提错误**

- `cur` 只表示当前结尾状态
- `ans` 才是全局答案

---

### 4）循环从 `i = 0` 开始
错因：**边界 + 逻辑错误**

第 0 个元素已经初始化过了，不能重复算。

---

### 5）不会解释为什么要“断开重新开始”
错因：**状态转移理解不清**

前面的和如果是负贡献，就会让当前结果变差，所以不如从当前位置重新开一段。

---

### 6）误以为必须开 `dp` 数组
错因：**空间优化没想到**

因为只依赖前一个状态，所以一个变量 `cur` 就够了。



## 8. 一句话记忆

**前面的和有用就接着加，前面的和没用就从当前重新开始。**
