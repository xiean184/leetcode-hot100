# LeetCode 19：删除链表的倒数第 N 个结点｜简短思路

## 1. 题目核心

- 给你一个单链表，删除链表的倒数第 `n` 个结点，并返回新的头结点。
- 这题不是子数组、子串、三元组、固定窗口、连续区间问题。
- 这是典型的 **链表删除结点 + 双指针** 问题。
- 核心限制：链表只能从前往后走，不能像数组一样直接访问倒数位置。

---

## 2. 核心知识点

- 单链表删除结点
- 虚拟头结点 `dummy`
- 快慢指针 / 双指针
- 快指针先走 `n` 步
- 慢指针停在“要删除结点的前一个结点”

---

## 3. 答题技巧

- 看到“删除链表某个结点”，要想到：**必须找到它的前一个结点**。
- 看到“倒数第 n 个”，要想到：链表不能倒着走，可以用 **快慢指针制造距离差**。
- 看到“可能删除头结点”，要想到：加一个 **虚拟头结点 `dummy`**。
- 看到“只遍历一次”，要想到：快指针先走，慢指针后走。

---

## 4. 最优思路

使用双指针：

1. 先创建虚拟头结点 `dummy`，让它指向 `head`。
2. 让 `fast` 和 `slow` 都从 `dummy` 出发。
3. `fast` 先走 `n` 步。
4. 然后 `fast` 和 `slow` 一起走。
5. 当 `fast` 走到最后一个结点时，`slow` 正好停在要删除结点的前一个结点。
6. 执行删除：

```cpp
slow->next = slow->next->next;
```

为什么成立？

因为 `fast` 比 `slow` 始终领先 `n` 个结点。  
当 `fast` 到链表尾部时，`slow` 后面刚好还有 `n` 个结点，所以 `slow->next` 就是倒数第 `n` 个结点。

---

## 5. 我遇到的疑问整理

### 疑问 1：暴力算法到底在干什么？

暴力算法的核心是：

先数出链表总长度，再把“倒数第 n 个”转换成“正数第几个”。

例如：

```text
1 -> 2 -> 3 -> 4 -> 5
n = 2
```

链表长度是 `5`。  
倒数第 `2` 个是正数第：

```text
5 - 2 + 1 = 4 个
```

也就是结点 `4`。

但是删除结点时，不能只找到 `4`，而是要找到它前面的结点 `3`。

所以代码中从 `dummy` 出发走：

```text
length - n
```

步，正好停在要删除结点的前一个结点。

---

### 疑问 2：为什么要找前一个结点？

因为链表删除不是直接“删掉当前结点”，而是改变前一个结点的指向。

例如要删除 `4`：

```text
1 -> 2 -> 3 -> 4 -> 5
```

要让 `3` 直接指向 `5`：

```text
1 -> 2 -> 3 -> 5
```

代码是：

```cpp
cur->next = cur->next->next;
```

所以 `cur` 必须指向要删除结点的前一个结点。

---

### 疑问 3：为什么需要 dummy？

因为可能删除头结点。

例如：

```text
head = [1], n = 1
```

删除后应该返回空链表。

如果不用 `dummy`，删除头结点需要单独判断。  
用了 `dummy` 后，删除头结点、删除中间结点、删除尾结点都可以统一处理。

---

### 疑问 4：为什么双指针里 fast 先走 n 步？

为了让 `fast` 和 `slow` 之间保持 `n` 个结点的距离。

这样当 `fast` 到达尾部时，`slow` 后面刚好还有 `n` 个结点，`slow->next` 就是要删除的倒数第 `n` 个结点。

---

### 疑问 5：为什么循环条件写 `fast->next != nullptr`？

因为我们希望 `slow` 停在“要删除结点的前一个结点”。

如果写成：

```cpp
while (fast != nullptr)
```

`slow` 可能会多走一步，导致位置错乱。

推荐写法：

```cpp
while (fast->next != nullptr)
```

---

## 6. 易错点

1. **忘记使用 dummy**
   - 删除头结点时容易出错。

2. **返回了 head**
   - 如果删除的是原头结点，`head` 已经不是新头结点。
   - 正确返回：

```cpp
return dummy->next;
```

3. **while 条件写错**
   - 推荐写：

```cpp
while (fast->next != nullptr)
```

4. **删除语句写错**
   - 正确写法：

```cpp
slow->next = slow->next->next;
```

   - 错误理解：

```cpp
slow = slow->next->next;
```

   这只是移动指针变量，没有改变链表连接关系。

5. **暴力法中搞不清 `length - n` 的含义**
   - 它不是要删除结点的位置。
   - 它表示：从 `dummy` 出发走几步，到达要删除结点的前一个结点。

## 7. 标准代码

```cpp
class Solution {
public:
    ListNode* removeNthFromEnd(ListNode* head, int n) {
        ListNode* dummy = new ListNode(0);
        dummy->next = head;

        ListNode* fast = dummy;
        ListNode* slow = dummy;

        for (int i = 0; i < n; i++) {
            fast = fast->next;
        }

        while (fast->next != nullptr) {
            fast = fast->next;
            slow = slow->next;
        }

        slow->next = slow->next->next;

        return dummy->next;
    }
};
```

## 8. 一句话记忆

删除链表倒数第 `n` 个结点：**dummy 统一边界，fast 先走 n 步，fast 和 slow 一起走到尾，slow 后面就是要删除的结点。**
