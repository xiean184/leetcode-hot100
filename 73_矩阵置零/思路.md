# 简短思路.md

## 73. 矩阵置零

---

## 1. 题目核心

给定一个 `m × n` 的矩阵，如果某个元素是 `0`，就把它所在的**整行和整列都变成 `0`**。

- 题型：**矩阵 + 标记法**
- 关键限制：**原地算法**
- 本质：**不能边扫描边直接清零，必须先标记，再统一处理**

---

## 2. 核心知识点

- **矩阵遍历**
- **标记法**
- **原地修改**
- **空间优化**

这题真正关键的不是“把 0 扩散出去”，而是：

**避免新产生的 0 影响后续判断。**

---

## 3. 答题技巧

### 看到这些信号，就想到“先标记，后处理”

| 题目信号 | 应想到的方法 |
|---|---|
| 某个位置影响整行整列 | 标记法 |
| 原地算法 | 用原矩阵做标记 |
| 不能污染后续判断 | 分两阶段处理 |
| 第一行第一列可复用 | 当作标记区 |

### 判断依据

如果题目要求：

- 遇到某些特殊值后，要批量修改一整行 / 一整列
- 又不能一边扫一边直接改
- 还要求原地操作

那么通常就要想到：

**先记录哪些行列需要处理，再统一修改。**

---

## 4. 最优思路

### 第一步：为什么不能直接改？

如果扫描到一个 `0` 就立刻把整行整列改成 `0`，  
那么你新改出来的 `0` 会被后续扫描误当成“原本就有的 0”，导致错误扩散。

所以这题必须分成两步：

1. **标记**
2. **清零**

---

### 第二步：如何原地标记？

最直接的方法是额外开：

- `row[m]`
- `col[n]`

但这不是最优。

更好的办法是：

- 用 **第一列** 记录哪一行将来要清零
- 用 **第一行** 记录哪一列将来要清零

如果某个位置 `matrix[i][j] == 0`，就做：

```cpp
matrix[i][0] = 0;
matrix[0][j] = 0;
```

---

### 第三步：为什么第一行和第一列要单独保存状态？

因为第一行和第一列既是“原始数据”，又要拿来当“标记区”，角色冲突。

所以要额外用两个变量记录：

```cpp
firstRowZero
firstColZero
```

分别表示：

- 第一行原本有没有 0
- 第一列原本有没有 0

---

### 第四步：处理顺序

1. 先检查第一行是否原本有 0
2. 再检查第一列是否原本有 0
3. 扫描内部区域，从 `(1,1)` 开始，遇到 0 就在第一行和第一列打标记
4. 根据第一列标记清零各行
5. 根据第一行标记清零各列
6. 最后再处理第一行和第一列

---

### 为什么这样成立

因为：

- 第一行存“列是否清零”
- 第一列存“行是否清零”
- 后面统一根据这些标记操作，就不会被中途修改污染

所以既能原地做，又能保证逻辑正确。

---

## 5. 我遇到的疑问整理

### 1）为什么不能一看到 0 就立刻整行整列清零？

因为会污染后续判断。

例如原本只有一个 0，  
你立刻改完后，矩阵里会多出很多新 0，  
这些新 0 不是原始信息，不能继续拿来扩散。

---

### 2）为什么要用第一行和第一列做标记？

因为题目要求原地算法，不能随便再开两个数组。

而第一行和第一列天然可以拿来复用，分别记录：

- 哪些列要清零
- 哪些行要清零

---

### 3）为什么第一行和第一列要单独用两个变量保存？

因为它们既是数据，又是标记区。

如果不额外保存：

- 第一行本来有 0 还是后来被标记成 0
- 第一列本来有 0 还是后来被标记成 0

你就分不清了。

---

### 4）为什么之前会越界报错？

因为你把：

- 行数 `m`
- 列数 `n`

混用了。

例如你写成：

```cpp
for (int j = 1; j < m; j++)
```

但 `j` 是列下标，应该和 `n` 比，不是和 `m` 比。

矩阵题里一定要记住：

- `i < m` 表示行
- `j < n` 表示列

否则在非方阵里很容易越界。

---

### 5）为什么内部扫描要从 `(1,1)` 开始？

因为第一行和第一列已经被拿去做标记区了。

所以做标记时不能把它们当普通区域再乱处理。

---

## 6. 易错点

### 1）一遇到 0 就直接清零
错因：**思路错误**

这样会把新产生的 0 当成原始 0，导致错误扩散。

---

### 2）忘记单独记录第一行和第一列
错因：**逻辑前提错误**

第一行和第一列同时承担“数据”和“标记”两种角色，不单独保存会混乱。

---

### 3）扫描内部区域时把第一行第一列也一起扫乱
错因：**边界错误**

内部扫描应该从 `(1,1)` 开始。

---

### 4）把 `m` 和 `n` 混用
错因：**越界错误**

- 行循环：`i < m`
- 列循环：`j < n`

非方阵下最容易炸。

---

### 5）最后忘记处理第一行或第一列
错因：**收尾不完整**

这是最常见漏点之一。
## 8. 一句话记忆

**不能边扫边清零，要先标记再处理；用第一行和第一列当标记区，再单独处理第一行第一列。**
