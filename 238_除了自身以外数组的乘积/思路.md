# 简短思路.md

## 238. 除自身以外数组的乘积

---

## 1. 题目核心

给定数组 `nums`，返回数组 `answer`，其中：

```cpp
answer[i] = nums 中除了 nums[i] 以外，其余所有元素的乘积
```

- 题型：**数组 + 前缀积/后缀积**
- 关键限制：**不能用除法**
- 时间要求：**O(n)**
- 本质：每个位置的答案都等于 **左边乘积 × 右边乘积**

---

## 2. 核心知识点

- **前缀积**
- **后缀积**
- **数组**
- **空间优化**

这题真正关键的不是重新枚举每个位置的其他元素，而是：

**把“除自己以外”的乘积拆成“左边所有数的乘积”和“右边所有数的乘积”。**

---

## 3. 答题技巧

### 看到这些信号，就想到“前缀 + 后缀”

| 题目信号 | 应想到的方法 |
|---|---|
| 每个位置都和左边、右边有关 | 前缀 / 后缀 |
| 不能用除法 | 不能走“总乘积 ÷ 自己” |
| 要求 O(n) | 两遍扫描 |
| 进阶要求 O(1) 额外空间 | 用答案数组存左积，再用变量维护右积 |

### 判断依据

如果题目要求：

- 对每个位置单独算答案
- 每个答案都依赖该位置左边和右边的信息
- 还不能暴力重复计算

那么通常就要想到：

**前缀信息 + 后缀信息。**

---

## 4. 最优思路

对于每个位置 `i`：

```cpp
answer[i] = 左边所有元素乘积 × 右边所有元素乘积
```

### 第一步：先算左积

定义：

```cpp
left[i] = nums[i] 左边所有元素的乘积
```

例如：

```text
nums = [1,2,3,4]
left = [1,1,2,6]
```

因为：

- `left[0] = 1`
- `left[1] = 1`
- `left[2] = 1*2 = 2`
- `left[3] = 1*2*3 = 6`

---

### 第二步：再算右积

定义：

```cpp
right[i] = nums[i] 右边所有元素的乘积
```

例如：

```text
right = [24,12,4,1]
```

因为：

- `right[3] = 1`
- `right[2] = 4`
- `right[1] = 3*4 = 12`
- `right[0] = 2*3*4 = 24`

---

### 第三步：合并答案

```cpp
answer[i] = left[i] * right[i]
```

---

### 第四步：再优化空间

其实没必要真的开 `left` 和 `right` 两个数组。

可以：

1. 先用 `answer` 数组直接存左积
2. 再用一个变量 `suffix` 从右往左维护右积
3. 一边走一边把右积乘到 `answer[i]` 上

这样时间 `O(n)`，额外空间 `O(1)`。

---

### 为什么这样成立

因为“除自己以外”的所有乘积，天然就能拆成：

- 自己左边那一段
- 自己右边那一段

而且左边和右边彼此独立，所以可以分开算，再乘起来。

---

## 5. 我遇到的疑问整理

### 1）为什么不能直接用总乘积除以自己？

因为题目明确要求：

**不能使用除法。**

而且遇到 `0` 时，这种思路也会很麻烦。

---

### 2）为什么答案一定是“左积 × 右积”？

因为除了 `nums[i]` 以外，剩下的元素正好就是两部分：

- `i` 左边所有元素
- `i` 右边所有元素

没有第三部分。

---

### 3）为什么前缀积要写成

```cpp
left[i] = left[i-1] * nums[i-1];
```

而不是：

```cpp
left[i] *= nums[i-1];
```

因为我们是在做“状态递推”。

`left[i]` 不是在原地乱乘，而是由前一个状态 `left[i-1]` 推出来的。

如果 `left[i]` 一开始是 0，再用 `*=`
，那么它会永远是 0。

---

### 4）为什么你之前代码会全是 0？

因为你写了类似：

```cpp
vector<int> left(n), right(n);
```

这会让数组默认初始化为全 0。

后面又写：

```cpp
left[i] *= ...
right[i] *= ...
```

那就是：

```text
0 × 任何数 = 0
```

所以最后全变成 0。

---

### 5）为什么后缀积不能从 `j = i` 开始乘？

因为 `right[i]` 的定义是：

**`i` 右边所有元素的乘积**

所以只能乘：

```cpp
nums[i+1], nums[i+2], ...
```

不能把 `nums[i]` 自己也乘进去。

---

### 6）为什么这题天然能处理 0？

因为我们根本没有用除法，而是老老实实算左边和右边的乘积。

所以即使数组里有 0，这种方法也照样成立，不需要额外特判。

---

## 6. 易错点

### 1）想当然用除法
错因：**思路错误**

题目明确禁止除法。

---

### 2）把前缀积数组初始化成 0 后再 `*=`
错因：**逻辑错误**

0 乘任何数都还是 0。

---

### 3）前缀积下标写错
错因：**边界错误**

正确写法：

```cpp
ans[i] = ans[i - 1] * nums[i - 1];
```

不是乘 `nums[i]`。

---

### 4）后缀积更新顺序写反
错因：**逻辑前提错误**

正确顺序：

```cpp
ans[i] *= suffix;
suffix *= nums[i];
```

不能先更新 `suffix` 再乘到 `ans[i]`。

---

### 5）把 `right[i]` 写成包含 `nums[i]` 自己
错因：**定义错误**

`right[i]` 只能表示右边，不包括自己。

---

### 6）复杂度写成 O(n^2)
错因：**方法不优**

题目要求 O(n)，不能对每个位置都重新扫一遍数组。



## 8. 一句话记忆

**每个位置的答案 = 左边乘积 × 右边乘积；先把左积存进答案，再从右往左乘上右积。**
