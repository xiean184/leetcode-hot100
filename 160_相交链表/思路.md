# 简短思路.md

## 160. 相交链表

---

## 1. 题目核心

给定两个单链表 `headA` 和 `headB`，找出它们**相交的起始节点**；如果不相交，返回 `nullptr`。

- 题型：**链表 + 双指针**
- 关键点：相交看的是**是否为同一个节点地址**
- 本质：让两个指针走过**相同总长度**，从而在交点相遇

---

## 2. 核心知识点

- **链表**
- **双指针**
- **路径对齐**
- **指针比较**

这题真正关键的不是比节点值，而是：

**比较两个指针是否指向同一个节点对象。**

---

## 3. 答题技巧

### 看到这些信号，就想到“双指针换头法”

| 题目信号 | 应想到的方法 |
|---|---|
| 两个链表找相交点 | 双指针 |
| 相交本质是公共尾部 | 让两指针走相同总路程 |
| 不想先算长度 | 换头法 |
| 判断相交 | 比较指针地址，不比值 |

### 判断依据

如果题目是：

- 两条链表
- 要找公共起点
- 不要求改链表
- 又想做到线性复杂度

那就优先想到：

**两个指针分别从两链表出发，走到头就换另一条链表继续走。**

---

## 4. 最优思路

设两个指针：

- `pA` 从 `headA` 出发
- `pB` 从 `headB` 出发

每次都往后走一步。

如果某个指针走到链表末尾，就让它跳到另一条链表头部继续走：

- `pA` 走完 A，就去走 B
- `pB` 走完 B，就去走 A

最后：

- 如果有交点，它们会在交点相遇
- 如果没有交点，它们会在 `nullptr` 相遇

---

### 为什么这样成立

假设：

- A 独有部分长度是 `a`
- B 独有部分长度是 `b`
- 公共部分长度是 `c`

那么：

- `pA` 总共走：`a + c + b`
- `pB` 总共走：`b + c + a`

两者总长度相同，所以最终会对齐。

---

## 5. 我遇到的疑问整理

### 1）为什么不能比较节点值？

因为题目里“相交”的定义是：

**两个指针指向同一个节点对象。**

也就是说要判断：

```cpp
pA == pB
```

而不是：

```cpp
pA->val == pB->val
```

值一样不代表是同一个节点。

---

### 2）为什么两个指针走到头后要换到另一条链表？

因为两条链表前面独有部分长度可能不同。  
通过“换头”，两个指针最终都能走完：

- 自己原来的链表
- 再加上对方的链表前半段

这样总路程就对齐了。

---

### 3）为什么不会死循环？

不会。

- 如果有交点，会在交点相遇
- 如果没有交点，两个指针最后都会变成 `nullptr`

一旦：

```cpp
pA == pB
```

循环就结束。

---

### 4）`ListNode* a` 和 `ListNode *a` 有什么区别？

**本质上没有区别。**

这两种写法在 C++ 里意思完全一样，都是：

```cpp
a 是一个指向 ListNode 的指针
```

也就是：

```cpp
ListNode* a;
```

和

```cpp
ListNode *a;
```

没有语义区别，只是书写风格不同。

---

### 5）那为什么很多人还会强调这个写法？

因为下面这种声明很容易让人误会：

```cpp
ListNode* a, b;
```

很多新手会误以为：

- `a` 是指针
- `b` 也是指针

其实不是。

真实含义是：

- `a` 是 `ListNode*`
- `b` 是普通 `ListNode`

因为星号是跟着变量名生效的，不是跟着类型名整体生效。

所以为了避免歧义，很多人更推荐这样写：

```cpp
ListNode *a;
ListNode *b;
```

一行只声明一个变量，更清楚。

---

## 6. 易错点

### 1）用节点值比较相交
错因：**思路错误**

必须比较地址：

```cpp
pA == pB
```

---

### 2）只让一个指针换头
错因：**逻辑错误**

必须两个指针都换，才能真正对齐路程。

---

### 3）担心没有交点会死循环
错因：**理解不扎实**

没有交点时，两个指针最终都会走到 `nullptr`，然后相等退出。

---

### 4）把 `ListNode* a, b;` 误认为两个都是指针
错因：**语法理解错误**

实际上只有 `a` 是指针，`b` 不是。

---

## 7. 标准代码

```cpp
class Solution {
public:
    ListNode *getIntersectionNode(ListNode *headA, ListNode *headB) {
        ListNode *pA = headA;
        ListNode *pB = headB;

        while (pA != pB) {
            pA = (pA == nullptr) ? headB : pA->next;
            pB = (pB == nullptr) ? headA : pB->next;
        }

        return pA;
    }
};
```

## 8. 一句话记忆

**两个指针分别走 A 和 B，走到头就换到对方头部，最后要么在交点相遇，要么一起到 `nullptr`。**
