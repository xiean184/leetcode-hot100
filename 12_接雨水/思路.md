# 接雨水_简短思路

## 核心知识点
- **按位置计算贡献**：不是直接看整张图，而是看每个位置自己能接多少水
- **核心公式**：`当前位置雨水 = min(leftMax, rightMax) - height[i]`
- **双指针**：`left` 从左往右，`right` 从右往左
- **动态维护左右最高值**：`leftMax`、`rightMax`
- **谁低处理谁**：哪一侧当前更低，就先确定哪一侧这一格的水量

---

## 答题技巧
1. 看到“每个位置的答案依赖左右信息”，先想到 `leftMax / rightMax`
2. 先记住公式：  
   `雨水 = 左右最高中的较小者 - 当前高度`
3. 如果不想开两个辅助数组，就用双指针边走边维护最大值
4. 双指针不是随便动，而是：  
   - 左边更低，处理左边  
   - 右边更低，处理右边

---

## 易错点
1. **不是看 `height[left]` 和 `height[right]` 谁高就直接加水**  
   真正决定接水量的是 `leftMax` 和 `rightMax`
2. **不是每次都能直接算左右两边**  
   只能先处理“当前较低的一侧”
3. **加水前先判断是否更新最大值**  
   如果当前柱子本身已经是新的最高柱子，就不能加水，只能更新 `leftMax` 或 `rightMax`
4. **不要忘记移动指针**
5. **雨水不可能是负数**  
   所以代码里先判断更新最大值，否则再加水

---

### 左边分支
```cpp
if (height[left] < height[right]) {
    if (height[left] >= leftMax) {
        leftMax = height[left];
    } else {
        ans += leftMax - height[left];
    }
    left++;
}
```

### 它到底在干什么
当 `height[left] < height[right]` 时，说明当前左边这格的水量可以先确定。

原因是：

- 右边至少有一根柱子比左边当前柱子高
- 左边这格最终能接多少水，关键看左边最高值 `leftMax`
- 这时左边这格的上限已经不会被右边当前柱子卡住

所以就分两种情况：

#### 情况 1：`height[left] >= leftMax`
说明当前左柱子已经和左边最高值一样高，甚至更高。

这表示：

- 这一格本身不能接水
- 它反而要成为新的“左边最高挡板”

所以写：

```cpp
leftMax = height[left];
```

#### 情况 2：`height[left] < leftMax`
说明当前柱子比左边最高挡板矮。

这时这一格可以接水，接水量就是：

```cpp
leftMax - height[left]
```

所以写：

```cpp
ans += leftMax - height[left];
```

最后：

```cpp
left++;
```

表示左边这一格已经处理完了，继续看下一格。

---

### 右边分支
```cpp
else {
    if (height[right] >= rightMax) {
        rightMax = height[right];
    } else {
        ans += rightMax - height[right];
    }
    right--;
}
```

逻辑和左边完全对称：

- 如果右边当前柱子已经不低于右边最高值，就更新 `rightMax`
- 否则这一格可以接水，接水量是：

```cpp
rightMax - height[right]
```

然后右指针左移。

---

## 一句话理解图片代码
> 左右两边谁更低，就先处理谁；处理这一边时，如果当前柱子刷新了最高值，就更新最高值，否则就按“最高值减当前高度”来接水。


## 一句话记忆
> 接雨水双指针的核心就是：谁低处理谁，谁这一侧的最高挡板更高，就能在这一格上存下差值那么多的水。
