# LeetCode 25：K 个一组翻转链表｜简短思路

## 1. 题目核心

- 给你一个单链表，每 `k` 个节点作为一组进行翻转，返回修改后的链表头结点。
- 如果最后剩余节点数量不足 `k` 个，则这一部分保持原顺序。
- 这题不是子数组、子串、三元组、固定窗口、连续区间问题。
- 这是典型的 **链表 + 分组反转 + 指针重连** 问题。
- 题目要求：不能只交换节点的值，必须真正改变节点之间的 `next` 指向。

---

## 2. 核心知识点

- 单链表反转
- 虚拟头结点 `dummy`
- 分组处理链表
- 每组反转前先判断是否够 `k` 个节点
- 标准反转链表模板：

```cpp
temp = cur->next;
cur->next = pre;
pre = cur;
cur = temp;
```

- 关键指针：
  - `groupPrev`：当前这一组的前一个节点
  - `kth`：当前这一组的第 `k` 个节点
  - `groupNext`：当前这一组后面的第一个节点
  - `pre`、`cur`、`temp`：标准反转链表三件套

---

## 3. 答题技巧

- 看到“每 k 个节点一组翻转”，要想到：**先分组，再反转每一组**。
- 看到“不足 k 个不翻转”，要想到：每次反转前必须先找第 `k` 个节点。
- 看到“头节点可能变化”，要想到：使用虚拟头结点 `dummy`。
- 看到“链表反转”，优先想到标准模板：

```cpp
temp = cur->next;
cur->next = pre;
pre = cur;
cur = temp;
```

- 普通反转链表是反转到 `nullptr` 停；这题是一组一组反转，所以反转到 `groupNext` 停。

---

## 4. 最优思路

使用 **标准反转链表模板法**。

整体步骤：

1. 创建虚拟头结点 `dummy`，指向原链表头 `head`。
2. 用 `groupPrev` 表示当前这一组的前一个节点。
3. 从 `groupPrev` 开始向后走 `k` 步，找到当前组的第 `k` 个节点 `kth`。
4. 如果找不到 `kth`，说明剩余节点不足 `k` 个，直接结束。
5. 记录当前组后面的第一个节点：

```cpp
groupNext = kth->next;
```

6. 用标准反转链表模板，把当前这一组从 `groupPrev->next` 反转到 `groupNext` 前面。
7. 反转完成后，把前面的链表接到新的组头 `kth`。
8. 把 `groupPrev` 移动到当前组反转后的尾巴，继续处理下一组。

为什么这样成立？

因为当前组是：

```text
groupPrev -> 组头 -> ... -> kth -> groupNext
```

反转后应该变成：

```text
groupPrev -> kth -> ... -> 组头 -> groupNext
```

其中：
- `kth` 是反转后的新组头；
- 原来的组头会变成反转后的组尾；
- 组尾必须接回 `groupNext`；
- 所以反转时要让 `pre = groupNext`。

## 5. 我遇到的疑问整理

### 疑问 1：头插法和标准反转模板法，哪个更好？

更推荐掌握 **标准反转模板法**。

原因：

- 它和 LeetCode 206「反转链表」是同一套思路；
- 更通用，后面很多链表题都能用；
- 比头插法更容易迁移到“反转链表 II”“回文链表”“重排链表”等题。

头插法代码也能做，但更像技巧；标准反转模板法更适合作为基础模板记忆。

---

### 疑问 2：为什么 `pre = groupNext`？

普通反转整条链表时：

```cpp
pre = nullptr;
```

因为最后一个节点反转后要指向空。

但这题只反转当前这一组。

例如：

```text
1 -> 2 -> 3 -> 4 -> 5
k = 3
```

只反转 `1,2,3`，结果应该是：

```text
3 -> 2 -> 1 -> 4 -> 5
```

所以反转后，`1` 不能指向 `nullptr`，而是要指向 `4`。

因此：

```cpp
pre = groupNext;
```

这里的 `groupNext` 就是这一组后面的第一个节点。

---

### 疑问 3：为什么循环条件是 `cur != groupNext`？

普通反转整条链表时：

```cpp
while (cur != nullptr)
```

因为要一直反转到链表结尾。

但这题只反转当前这一组，不能碰下一组。

所以写：

```cpp
while (cur != groupNext)
```

意思是：

```text
只反转当前组，反转到 groupNext 前面就停止。
```

---

### 疑问 4：为什么反转后还要接回去？

反转一组后，组内指针已经反过来了，但前面的链表还没有接到新的组头。

例如：

```text
dummy -> 1 -> 2 -> 3 -> 4 -> 5
```

反转 `1,2,3` 后，组内变成：

```text
3 -> 2 -> 1 -> 4 -> 5
```

但是 `dummy->next` 还需要改成 `3`。

所以要写：

```cpp
ListNode* oldGroupHead = groupPrev->next;
groupPrev->next = kth;
groupPrev = oldGroupHead;
```

含义：

- `oldGroupHead` 是原组头，反转后会变成组尾；
- `groupPrev->next = kth` 把前面接到新的组头；
- `groupPrev = oldGroupHead` 移动到当前组尾部，准备处理下一组。

---

### 疑问 5：为什么我之前会空指针报错？

主要原因是循环次数或边界判断不对。

常见错误：

- 没有先判断剩余节点是否够 `k` 个；
- 反转时多反转了下一组节点；
- `second` 或 `cur` 已经是 `nullptr`，还访问了 `second->next` 或 `cur->next`；
- 指针更新顺序错误，导致链表断开或成环。

这类错误属于：

```text
边界错误 + 越界错误 + 指针维护错误
```

## 6. 易错点

1. **剩余节点不足 k 个也反转了**

题目要求：

```text
不足 k 个节点保持原顺序
```

所以每组反转前必须先找到第 `k` 个节点。

---

2. **忘记使用 dummy**

第一组反转后，头节点可能发生变化。

例如：

```text
1 -> 2 -> 3
```

反转后：

```text
3 -> 2 -> 1
```

新头不再是 `1`，所以要返回：

```cpp
dummy->next
```

---

3. **`pre` 初始化错**

标准反转当前组时应该写：

```cpp
pre = groupNext;
```

不是：

```cpp
pre = nullptr;
```

否则当前组尾巴无法接回后面的链表。

---

4. **循环条件写错**

正确写法：

```cpp
while (cur != groupNext)
```

不要写成：

```cpp
while (cur != nullptr)
```

否则会把后面的节点也一起反转。

---

5. **忘记保存旧组头**

反转后需要移动 `groupPrev` 到当前组尾巴，所以要先保存：

```cpp
ListNode* oldGroupHead = groupPrev->next;
```

---

6. **忘记移动 groupPrev**

一组反转完成后要写：

```cpp
groupPrev = oldGroupHead;
```

否则无法正确处理下一组。

---

## 7. 标准代码

```cpp
class Solution {
public:
    ListNode* reverseKGroup(ListNode* head, int k) {
        ListNode* dummy = new ListNode(0);
        dummy->next = head;

        ListNode* groupPrev = dummy;

        while (true) {
            ListNode* kth = groupPrev;

            for (int i = 0; i < k && kth != nullptr; i++) {
                kth = kth->next;
            }

            if (kth == nullptr) {
                break;
            }

            ListNode* groupNext = kth->next;

            ListNode* pre = groupNext;
            ListNode* cur = groupPrev->next;

            while (cur != groupNext) {
                ListNode* temp = cur->next;
                cur->next = pre;
                pre = cur;
                cur = temp;
            }

            ListNode* oldGroupHead = groupPrev->next;
            groupPrev->next = kth;
            groupPrev = oldGroupHead;
        }

        return dummy->next;
    }
};
```

---

## 8. 一句话记忆

K 个一组翻转链表：**先用 kth 判断够不够 k 个，够就用 pre / cur / temp 反转到 groupNext 前面，再把 groupPrev 接到 kth，并移动到旧组头。**
