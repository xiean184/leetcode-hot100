# 找到字符串中所有字母异位词_简短思路

## 题目核心
给定两个字符串 `s` 和 `p`，在 `s` 中找到所有和 `p` 互为**字母异位词**的子串，并返回这些子串的起始下标。

注意：

- 异位词看的是**字符种类和出现次数**
- 不看字符顺序
- 子串必须是**连续的**
- 窗口长度固定为 `p.size()`

---

## 核心知识点
- **滑动窗口**
  - 因为要找的是 `s` 中所有长度等于 `p.size()` 的连续子串
- **计数数组**
  - 用两个长度为 26 的数组分别统计：
    - `p` 中每个字母出现次数
    - 当前窗口中每个字母出现次数
- **字符映射到数组下标**
  - 通过：
    ```cpp
    c - 'a'
    ```
    把小写字母映射到 `0~25`
- **固定长度窗口维护**
  - 右边进一个字符，左边出一个字符

---

## 答题技巧
1. 看到“**异位词**”，先想到“**字符次数相同**”，不是顺序相同  
2. 看到“**所有子串**”且长度固定，优先想到 **滑动窗口**  
3. 题目限定只有小写字母时，优先用长度为 26 的数组，比 `unordered_map` 更简单、更快  
4. 窗口长度固定后，每次只需要：
   - 右边加一个字符
   - 左边减一个字符
   - 再比较两个频次数组是否相同

---

## 最优思路
用两个数组：

```cpp
vector<int> cntP(26, 0), cntS(26, 0);
```

其中：

- `cntP[i]`：记录 `p` 中第 `i` 个字母出现次数
- `cntS[i]`：记录当前窗口中第 `i` 个字母出现次数

步骤如下：

1. 先统计 `p` 的字符频次
2. 用右指针遍历 `s`
3. 每进来一个字符，就更新当前窗口频次
4. 如果窗口长度超过 `p.size()`，就把最左边字符移出去
5. 当窗口长度正好等于 `p.size()` 时，比较 `cntS` 和 `cntP`
6. 如果相同，说明当前窗口是异位词，记录起始下标

## 你遇到的疑问整理

### 问题 1：为什么不能只排序 `p`？
因为只排序 `p` 没有意义。  
如果你想靠排序判断异位词，就必须把 `s` 的每个窗口也都排序后再比较，这样会很慢。

所以更好的办法不是排序，而是直接比较**字符次数**。

---

### 问题 2：为什么不能用 `unordered_set<char>`？
因为 `set` 只能告诉你：

> 某个字符在不在

但这题需要知道的是：

> 某个字符出现了几次

例如：

```text
p = "aab"
```

必须要区分：

- `a` 出现 2 次
- `b` 出现 1 次

而 `set` 只能表示：

```text
{'a', 'b'}
```

分不出 1 个 `a` 还是 2 个 `a`，所以不适合这题。

---

### 问题 3：为什么这样定义？
```cpp
vector<int> cntP(26, 0), cntS(26, 0);
```

因为题目里只有小写字母 `a~z`，一共 26 个。

所以可以开两个长度为 26 的整数数组：

- 一个统计 `p`
- 一个统计当前窗口

每个位置对应一个字母：

- 下标 0 表示 `a`
- 下标 1 表示 `b`
- ...
- 下标 25 表示 `z`

数组里的值表示：

> 这个字母出现了几次

例如：

```cpp
cntP['c' - 'a']++;
```

就是把 `c` 对应位置的次数加 1。

---

### 问题 4：为什么要用固定长度窗口？
因为异位词子串的长度必须和 `p` 一样。

如果 `p` 长度是 3，那你只需要检查 `s` 中所有长度为 3 的窗口。

所以窗口大小始终固定为：

```cpp
p.size()
```

---

## 易错点
1. **把异位词理解成顺序相同**
   - 实际上只要求字符种类和次数相同
2. **误用 `unordered_set<char>`**
   - 这题要统计次数，不是只判断出现没出现
3. **只顾右边扩张，不顾左边缩小**
   - 这题窗口长度必须固定
4. **不会理解 `c - 'a'`**
   - 它是把字符映射到 `0~25` 的下标
5. **窗口起点算错**
   - 当前窗口起点应该是：
     ```cpp
     right - m + 1
     ```


## 一句话记忆
> 这题本质就是：用固定长度滑动窗口维护当前子串的字符频次，并和 `p` 的字符频次比较。
