# LeetCode 24：两两交换链表中的节点｜简短思路

## 1. 题目核心

- 给你一个单链表，把链表中的节点 **两两交换**，并返回交换后的头结点。
- 例如：

```text
1 -> 2 -> 3 -> 4
```

交换后变成：

```text
2 -> 1 -> 4 -> 3
```

- 这题不是子数组、子串、三元组、固定窗口、连续区间问题。
- 这是典型的 **链表 + 指针重连 + 虚拟头结点** 问题。
- 题目要求：不能只交换节点的值，必须真正交换节点本身，也就是改变 `next` 指向。

---

## 2. 核心知识点

- 单链表指针操作
- 虚拟头结点 `dummy`
- 每次处理两个相邻节点
- 三指针：
  - `cur`：当前要交换的两个节点的前一个节点
  - `first`：当前组第一个节点
  - `second`：当前组第二个节点
- 原地修改链表，空间复杂度为 `O(1)`

---

## 3. 答题技巧

- 看到“链表中两两交换节点”，要想到：**不是交换值，而是改指针连接关系**。
- 看到“头两个节点也可能被交换”，要想到：使用 **虚拟头结点 `dummy`**。
- 看到“每次交换两个相邻节点”，要想到：让 `cur` 永远站在当前组前面。
- 看到“链表指针容易乱”，要先保存关键节点：

```cpp
ListNode* first = cur->next;
ListNode* second = cur->next->next;
```

---

## 4. 最优思路

使用原地指针交换。

每次把这一段：

```text
cur -> first -> second -> 后面
```

改成：

```text
cur -> second -> first -> 后面
```

固定三句：

```cpp
first->next = second->next;
second->next = first;
cur->next = second;
```

交换完成后，`first` 已经变成这一组的尾巴。  
下一组要从 `first` 后面开始，所以：

```cpp
cur = first;
```

为什么这样成立？

因为链表节点本身没有移动，只是改变节点之间的 `next` 指向。  
每次交换两个节点后，把 `cur` 移到这一组的尾部，就能继续处理下一组。

---

## 5. 我遇到的疑问整理

### 疑问 1：这题到底是不是交换地址？

不是。

这题不是把两个节点的内存地址互换，也不是交换节点的值。  
本质是：

```text
改变节点之间的 next 箭头。
```

例如原来：

```text
1 -> 2 -> 3 -> 4
```

交换 `1` 和 `2` 后：

```text
2 -> 1 -> 3 -> 4
```

节点本身没有消失，只是箭头变了。

---

### 疑问 2：为什么不能直接交换 val？

题目要求必须交换节点本身，而不是只交换节点中的值。

错误写法：

```cpp
swap(first->val, second->val);
```

虽然输出值可能看起来对，但不符合题意。

---

### 疑问 3：为什么要用 dummy？

因为头两个节点也要交换。

例如：

```text
1 -> 2 -> 3 -> 4
```

交换后头节点变成 `2`。

如果没有 `dummy`，处理头节点变化会比较麻烦。  
用了 `dummy` 后，链表统一变成：

```text
dummy -> 1 -> 2 -> 3 -> 4
```

交换后：

```text
dummy -> 2 -> 1 -> 3 -> 4
```

最后返回：

```cpp
dummy->next;
```

即可。

---

### 疑问 4：三句交换代码为什么这样写？

交换前：

```text
cur -> first -> second -> nextPart
```

目标：

```text
cur -> second -> first -> nextPart
```

第一句：

```cpp
first->next = second->next;
```

意思是：先让 `first` 接到 `second` 后面的部分，避免后面形成环。

第二句：

```cpp
second->next = first;
```

意思是：让 `second` 指向 `first`，完成两个节点内部反转。

第三句：

```cpp
cur->next = second;
```

意思是：让前面的 `cur` 接到新的头 `second` 上。

可以记成：

```text
拆 -> 反 -> 接
```

---

### 疑问 5：为什么交换后写 `cur = first`？

交换前：

```text
cur -> first -> second -> 后面
```

交换后：

```text
cur -> second -> first -> 后面
```

交换后，`first` 变成了这一组的最后一个节点。  
下一组要从 `first` 后面开始，所以 `cur` 应该移动到 `first`。

---

### 疑问 6：为什么我之前会超时？

你之前代码的问题主要是链表指针改乱了，可能形成了环。

常见错误包括：

- `dummy = head`，这不是真正的虚拟头结点
- `while (cur != nullptr)`，没有保证后面至少有两个节点
- 没有正确移动 `cur`
- 用 `head` 和 `cur->next` 混着改，导致链表结构被破坏
- 可能形成类似 `1 -> 2 -> 1 -> 2 ...` 的环，所以运行超时

---

## 6. 易错点

1. **把交换节点写成交换值**

```cpp
swap(first->val, second->val);
```

题目不允许。

---

2. **dummy 写错**

错误写法：

```cpp
ListNode* dummy = head;
```

正确写法：

```cpp
ListNode* dummy = new ListNode(0);
dummy->next = head;
```

---

3. **while 条件写错**

错误写法：

```cpp
while (cur != nullptr)
```

正确写法：

```cpp
while (cur->next != nullptr && cur->next->next != nullptr)
```

原因：必须保证后面至少有两个节点才能交换。

---

4. **交换指针顺序写乱**

推荐固定记忆：

```cpp
first->next = second->next;
second->next = first;
cur->next = second;
```

不要随便调换顺序，否则容易断链或成环。

---

5. **交换后 cur 移动错**

正确写法：

```cpp
cur = first;
```

因为交换后 `first` 是当前组的尾巴，也是下一组的前一个节点。

---

6. **返回值写错**

不要返回：

```cpp
return head;
```

因为原来的 `head` 可能已经不是新头节点。

正确返回：

```cpp
return dummy->next;
```

---

## 7. 标准代码

```cpp
class Solution {
public:
    ListNode* swapPairs(ListNode* head) {
        ListNode* dummy = new ListNode(0);
        dummy->next = head;

        ListNode* cur = dummy;

        while (cur->next != nullptr && cur->next->next != nullptr) {
            ListNode* first = cur->next;
            ListNode* second = cur->next->next;

            first->next = second->next;
            second->next = first;
            cur->next = second;

            cur = first;
        }

        return dummy->next;
    }
};
```

---

## 8. 一句话记忆

两两交换链表节点：**dummy 开头，cur 站在每组前面，保存 first 和 second，按“拆、反、接”三句改指针，最后 cur 移到 first。**
