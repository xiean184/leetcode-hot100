# 01 两数之和

## 题意
给定一个整数数组 nums 和目标值 target，找出和为 target 的两个元素下标。

## 暴力解法
使用两层循环，枚举所有两个数的组合。
如果 nums[i] + nums[j] == target，就返回这两个下标。

## 为什么慢
因为要枚举所有组合，时间复杂度是 O(n^2)。
当数组很大时，效率会比较低。

## 优化思路
使用哈希表。
一边遍历数组，一边检查 target - nums[i] 是否已经出现过。
如果出现过，就直接返回。
如果没出现过，就把当前数字和下标存入哈希表。

## 复杂度
- 暴力解法：时间复杂度 O(n^2)，空间复杂度 O(1)
- 哈希表解法：时间复杂度 O(n)，空间复杂度 O(n)

## 易错点
- 返回的是下标，不是数值
- 哈希表里存的是“数值 -> 下标”
- 先判断 need 是否存在，再存当前元素
