# 简短思路.md

## 48. 旋转图像

---

## 1. 题目核心

给定一个 `n × n` 的矩阵 `matrix`，要求把它**原地顺时针旋转 90°**。

- 题型：**矩阵 + 坐标变换 + 原地操作**
- 关键限制：**不能新开一个同样大小的矩阵**
- 本质：顺时针旋转 90°，可以拆成 **先转置，再每一行翻转**

---

## 2. 核心知识点

- **矩阵转置**
- **行翻转**
- **原地修改**
- **坐标变换**

这题真正关键的不是死记公式，而是理解：

**顺时针旋转 90° = 转置 + 每一行左右反转。**

---

## 3. 答题技巧

### 看到这些信号，就想到“转置 + 翻转”

| 题目信号 | 应想到的方法 |
|---|---|
| 方阵旋转 90° | 转置 + 翻转 |
| 要求原地 | 不能开新矩阵 |
| 顺时针旋转 | 转置后每一行翻转 |
| 逆时针旋转 | 转置后每一列翻转 / 先上下翻转再转置 |

### 判断依据

如果题目要求：

- 对一个方阵做旋转
- 而且要求原地完成
- 旋转角度是 90°

那就优先想到：

**先沿主对角线转置，再把每一行翻过来。**

---

## 4. 最优思路

### 第一步：先转置

转置的含义是：

```cpp
matrix[i][j] <-> matrix[j][i]
```

也就是把矩阵沿主对角线翻折。

例如：

```text
1 2 3
4 5 6
7 8 9
```

转置后变成：

```text
1 4 7
2 5 8
3 6 9
```

---

### 第二步：每一行左右翻转

对转置后的每一行再做一次反转：

```text
1 4 7 -> 7 4 1
2 5 8 -> 8 5 2
3 6 9 -> 9 6 3
```

最终得到：

```text
7 4 1
8 5 2
9 6 3
```

这正是顺时针旋转 90° 的结果。

---

### 为什么这样成立

顺时针旋转 90° 的坐标变化是：

```cpp
(i, j) -> (j, n - 1 - i)
```

而“转置 + 每行翻转”刚好等价于这个变化：

1. 转置：
   ```cpp
   (i, j) -> (j, i)
   ```

2. 每行翻转：
   ```cpp
   (j, i) -> (j, n - 1 - i)
   ```

所以两步合起来，正好就是顺时针旋转 90°。

---

## 5. 我遇到的疑问整理

### 1）为什么不能直接开一个新矩阵？

因为题目要求：

**必须原地旋转。**

新开一个 `n × n` 矩阵虽然容易写，但额外空间复杂度是 `O(n^2)`，不符合要求。

---

### 2）为什么转置时不能把整个矩阵都交换一遍？

因为会交换两次。

例如 `(0,1)` 和 `(1,0)`：

- 第一次交换过去
- 第二次又交换回来

所以转置时只需要处理主对角线**上方**或**下方**的一半。

常见写法是：

```cpp
for (int i = 0; i < n; i++) {
    for (int j = i + 1; j < n; j++) {
        swap(matrix[i][j], matrix[j][i]);
    }
}
```

---

### 3）为什么这题是“转置 + 每行翻转”，不是“转置 + 每列翻转”？

因为题目要求的是：

**顺时针旋转 90°**

- 顺时针：转置 + 每一行翻转
- 逆时针：转置 + 每一列翻转（或先上下翻转再转置）

方向不能搞混。

---

### 4）为什么这里是方阵才能这样做？

因为题目给的是：

```text
n × n
```

只有方阵旋转 90° 后，维度才仍然和原来一样，才能原地操作。

如果是普通 `m × n` 矩阵，旋转后维度会变化，就不能直接这样原地做。

---

## 6. 易错点

### 1）直接开新矩阵
错因：**空间不符合要求**

能做，但不满足“原地”。

---

### 2）转置时内层循环从 `0` 开始
错因：**逻辑错误**

会导致交换两次，最后又换回去。

正确写法：

```cpp
for (int j = i + 1; j < n; j++)
```

---

### 3）把顺时针和逆时针搞混
错因：**思路错误**

这题是：

- **顺时针 90°**
- 所以必须是 **转置 + 每行翻转**

---

### 4）把“每行翻转”误写成“上下翻转”
错因：**操作混淆**

顺时针旋转不是简单上下颠倒，而是每一行内部左右反转。

---

### 5）只会背公式，不会手推样例
错因：**理解不扎实**

这题如果不手推一遍 `3×3` 的样例，很容易把变换方向记混。


## 8. 一句话记忆

**顺时针旋转 90° = 先转置，再把每一行左右翻转。**
