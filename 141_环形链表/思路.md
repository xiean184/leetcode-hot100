# 环形链表_简短思路

## 题目核心
给定链表头节点 `head`，判断链表中是否存在环。

也就是：

- 如果顺着 `next` 一直走，最后能走到 `nullptr`，说明**无环**
- 如果后面某个节点又能回到前面某个节点，说明**有环**

---

## 核心知识点
- **链表遍历**
- **快慢指针（Floyd 判圈法）**
- **判空保护**
  - 快指针每次走两步前，必须保证：
    ```cpp
    fast != nullptr && fast->next != nullptr
    ```

---

## 答题技巧
1. 看到“**链表是否有环**”，优先想到 **快慢指针**
2. 慢指针每次走 1 步，快指针每次走 2 步
3. 如果有环，快指针一定会在环里追上慢指针
4. 如果无环，快指针一定会先走到空指针

---

## 最优思路
定义两个指针：

- `slow`：一次走一步
- `fast`：一次走两步

遍历时：

- 如果 `slow == fast`，说明有环，返回 `true`
- 如果 `fast == nullptr` 或 `fast->next == nullptr`，说明无环，返回 `false`

---

## 你写代码时遇到的问题整理

### 问题 1：为什么我一开始就返回 `true` 了？
因为你一开始写的是：

```cpp
ListNode* slow = head;
ListNode* fast = head;
```

此时：

```cpp
slow == fast
```

天然成立。

如果你把循环写成：

```cpp
while (slow != fast)
```

那循环根本不会进入，程序会直接往后执行，从而误判有环。

所以这题必须：

> **先移动，再比较**

---

### 问题 2：为什么只判断 `slow != nullptr && fast != nullptr` 不够？
因为你后面写了：

```cpp
fast = fast->next->next;
```

这要求：

- `fast` 不是空
- `fast->next` 也不是空

所以只判断 `fast != nullptr` 不够，还必须判断：

```cpp
fast->next != nullptr
```

否则会访问空指针。

---

### 问题 3：为什么比较的是节点地址，不是节点值？
因为链表里不同节点可能值一样。

这题要判断的是：

> 两个指针是不是指向同一个节点

所以必须写：

```cpp
if (slow == fast)
```

不能写成：

```cpp
if (slow->val == fast->val)
```

---

## 易错点
1. **把初始 slow=head、fast=head 的重合误判成相遇**
2. **忘记检查 `fast->next` 是否为空**
3. **把节点地址相等误写成节点值相等**
4. **把“先比较再移动”写反**
   - 正确是：**先移动，再判断是否相遇**

---

## 标准代码
```cpp
class Solution {
public:
    bool hasCycle(ListNode *head) {
        ListNode* slow = head;
        ListNode* fast = head;

        while (fast != nullptr && fast->next != nullptr) {
            slow = slow->next;
            fast = fast->next->next;

            if (slow == fast) {
                return true;
            }
        }

        return false;
    }
};
```

---

## 一句话记忆
> 环形链表的标准做法是快慢指针：慢指针走一步，快指针走两步；先走再比较，能相遇就有环，快指针走到空就无环。
