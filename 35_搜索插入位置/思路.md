# LeetCode 35：搜索插入位置｜简短思路

## 1. 题目核心

- 给定一个 **升序数组** `nums` 和目标值 `target`。
- 如果 `target` 在数组中，返回它的下标。
- 如果 `target` 不在数组中，返回它应该被插入的位置。
- 这题不是子数组、子串、三元组、滑动窗口问题。
- 这是典型的 **有序数组 + 二分查找** 问题。
- 题目明确要求时间复杂度为 `O(log n)`，所以应该使用二分查找。

---

## 2. 核心知识点

- 二分查找
- 有序数组
- 查找第一个大于等于 `target` 的位置
- 闭区间写法：`[left, right]`
- 边界处理：
  - `target` 比所有数小，返回 `0`
  - `target` 比所有数大，返回 `nums.size()`

---

## 3. 答题技巧

- 看到“有序数组”并且要求 `O(log n)`，立刻想到 **二分查找**。
- 看到“如果不存在，返回插入位置”，要想到：找 **第一个大于等于 target 的位置**。
- 如果 `nums[mid] >= target`，说明 `mid` 可能是答案，但左边可能还有更靠前的位置，所以继续往左找。
- 如果 `nums[mid] < target`，说明 `mid` 和左边都太小，应该往右找。

---

## 4. 最优思路

使用二分查找。

目标不是单纯判断 `target` 是否存在，而是找：

```text
第一个大于等于 target 的下标
```

原因：

- 如果 `target` 存在，这个位置就是 `target` 的下标。
- 如果 `target` 不存在，这个位置就是它应该插入的位置。
- 如果所有元素都小于 `target`，说明它应该插入到最后，返回 `nums.size()`。

二分过程：

```text
left = 0
right = nums.size() - 1
ans = nums.size()
```

每次取中间位置 `mid`：

- 如果 `nums[mid] >= target`：
  - `mid` 可能是答案；
  - 记录 `ans = mid`；
  - 继续向左找：`right = mid - 1`。
- 如果 `nums[mid] < target`：
  - 说明当前位置太小；
  - 向右找：`left = mid + 1`。

## 5. 我遇到的疑问整理

### 疑问 1：为什么不是普通遍历？

普通遍历可以做，但时间复杂度是 `O(n)`。

题目要求：

```text
O(log n)
```

所以必须使用二分查找。

---

### 疑问 2：为什么找的是第一个大于等于 target 的位置？

因为插入位置的本质是：

```text
target 应该放在第一个不小于它的数前面
```

例如：

```text
nums = [1,3,5,6], target = 2
```

第一个大于等于 `2` 的数是 `3`，下标是 `1`。

所以 `2` 应该插入到下标 `1`。

---

### 疑问 3：为什么 `ans` 初始值是 `nums.size()`？

因为如果 `target` 比所有元素都大，就应该插入到数组最后。

例如：

```text
nums = [1,3,5,6], target = 7
```

插入位置是下标 `4`，也就是：

```cpp
nums.size()
```

所以先默认答案是 `nums.size()`。

---

### 疑问 4：为什么 `nums[mid] >= target` 时还要继续往左找？

因为 `mid` 虽然可能是答案，但不一定是最靠左的位置。

我们要找的是：

```text
第一个大于等于 target 的位置
```

所以找到一个满足条件的位置后，还要继续往左缩小范围。

---

### 疑问 5：为什么循环条件是 `left <= right`？

这份代码使用的是闭区间写法：

```text
[left, right]
```

只要 `left <= right`，说明区间中还有元素可以查找。

当 `left > right` 时，说明搜索结束。

---

## 6. 易错点

1. **返回值搞错**

返回的是下标，不是目标值，也不是插入后的数组。

---

2. **忘记 target 比所有元素都大的情况**

例如：

```text
nums = [1,3,5,6], target = 7
```

应该返回：

```text
4
```

所以 `ans` 初始值要设为：

```cpp
nums.size()
```

---

3. **边界更新写错**

闭区间写法中：

```cpp
while (left <= right)
```

对应更新是：

```cpp
right = mid - 1;
left = mid + 1;
```

不要写混。

---

4. **不理解为什么判断条件是 `>=`**

因为这题找的是插入位置，也就是第一个大于等于 `target` 的位置。

---

5. **mid 写法不规范**

推荐写：

```cpp
int mid = left + (right - left) / 2;
```

不要写：

```cpp
int mid = (left + right) / 2;
```

虽然本题数据范围不大，但前一种写法更安全，可以避免极端情况下整数溢出。

---

## 7. 标准代码

```cpp
class Solution {
public:
    int searchInsert(vector<int>& nums, int target) {
        int left = 0;
        int right = nums.size() - 1;
        int ans = nums.size();

        while (left <= right) {
            int mid = left + (right - left) / 2;

            if (nums[mid] >= target) {
                ans = mid;
                right = mid - 1;
            } else {
                left = mid + 1;
            }
        }

        return ans;
    }
};
```

## 8. 一句话记忆

搜索插入位置：**在升序数组中，用二分查找第一个大于等于 `target` 的下标；如果找不到，就返回 `nums.size()`。**
