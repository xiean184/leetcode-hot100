# 三数之和_简短思路

## 题目核心
在数组中找出所有满足：

```
a + b + c = 0
```

的三元组，并且：

- 三个下标不能相同
- 答案中**不能有重复三元组**

---

## 暴力思路为什么不行
最直接的方法是三层循环枚举 `i、j、k`。

### 问题 1：太慢
三层循环复杂度是：

O(n^3)

数据一大就容易超时。

### 问题 2：会有重复答案
例如：

```cpp
[-1,0,1]
[0,1,-1]
```

虽然顺序不同，但本质上是同一个三元组，题目不允许重复。

---

## 最优思路
使用：

> **排序 + 固定一个数 + 双指针**

步骤如下：

1. 先对数组排序
2. 枚举第一个数 `nums[i]`
3. 在后面的区间里，用双指针 `left` 和 `right` 找另外两个数
4. 根据三数之和的大小移动指针：
   - 如果和小于 0，`left++`
   - 如果和大于 0，`right--`
   - 如果和等于 0，记录答案，然后跳过重复值

---

## 为什么一定要排序
排序有两个作用：

### 1. 方便双指针
排序后，才能通过“和太小/和太大”来决定移动哪边。

### 2. 方便去重
相同的数会挨在一起，更容易跳过重复答案。

---

## 双指针的核心逻辑
固定 `nums[i]` 后，问题变成：

```cpp
nums[left] + nums[right] = -nums[i]
```

所以：

- `left = i + 1`
- `right = nums.size() - 1`

然后在有序区间里逼近。

---

## 你写代码时遇到的主要问题整理

### 问题 1：返回类型写错
你一开始写过：

```cpp
int ans[3];
return ans;
```

但函数要求返回的是：

```cpp
vector<vector<int>>
```

也就是“很多组三元组”，不是一个长度为 3 的普通数组。

---

### 问题 2：暴力法没有去重
你后来改成了 `vector<vector<int>> ans;`，但还是会输出：

```cpp
[[-1,0,1],[-1,2,-1],[0,1,-1]]
```

其中：

```cpp
[-1,0,1]
[0,1,-1]
```

其实是重复的，只是顺序不同。

这说明：

> 三数之和这题不能只判断和为 0，还必须处理重复三元组。

---

### 问题 3：你写的“伪双指针”不是真正的双指针
你写过类似：

```cpp
int j = i + 1;
int k = j + 1;
while (nums[i] + nums[j] + nums[k] != 0) {
    j++;
    k++;
}
```

这不对。

因为真正的双指针应该是：

- 一个从左边走
- 一个从右边走

也就是：

```cpp
int left = i + 1;
int right = nums.size() - 1;
```

而不是 `j` 和 `k` 一起往右冲。

---

### 问题 4：没有边界判断，导致越界
你那个 while 写法没有控制：

```cpp
left < right
```

所以指针会一直增加，最后访问到数组外面，先越界，再触发运行时错误。

表面报的是：

```text
signed integer overflow
```

但根本原因其实是：

> **数组越界访问**

---

### 问题 5：找到答案后没有正确去重
三数之和这题最容易错的地方就是：

- `i` 要去重
- 找到答案后，`left` 要去重
- 找到答案后，`right` 也要去重

否则就会重复加入答案。

---

## 这题最该记住的点

### 1. 不是暴力三层循环硬做
而是：

> **固定一个数，把问题降成两数之和**

### 2. 不是随便双指针
而是：

- `left` 从左边开始
- `right` 从右边开始
- 根据 `sum` 的大小决定移动谁

### 3. 这题真正难点不是求和，而是去重
去重没做好，就一定错。

---

## 一句话记忆
> **三数之和 = 排序 + 固定一个数 + 双指针找另外两个数 + 全程去重。**
