# 简短思路.md

## 206. 反转链表

---

## 1. 题目核心

给定单链表头节点 `head`，要求把链表**反转**，并返回反转后的新头节点。

- 题型：**链表 + 指针操作**
- 关键点：改变的是 **next 指针方向**
- 本质：让当前节点指向前一个节点

---

## 2. 核心知识点

- **单链表**
- **三指针**
- **指针修改顺序**
- **原地反转**

这题真正关键的不是值交换，而是：

**正确修改每个节点的 `next` 指针，同时不能把后面的链表弄丢。**

---

## 3. 答题技巧

### 看到这些信号，就想到“三指针反转”

| 题目信号 | 应想到的方法 |
|---|---|
| 反转链表 | 三指针 |
| 改变 next 方向 | `prev / cur / nxt` |
| 不能丢链 | 先保存 `cur->next` |
| 原地操作 | 不新开链表，直接改指针 |

### 判断依据

如果题目要求：

- 链表整体反转
- 改的是节点连接方向
- 不是重建新链表

那么优先想到：

**当前节点反转时，要先记住后继，再改 `next`。**

---

## 4. 最优思路

用三个指针：

- `prev`：已经反转好的前半段头节点
- `cur`：当前正在处理的节点
- `nxt`：暂存 `cur` 的后继节点

每次循环做四件事：

1. 先保存下一个节点
2. 当前节点指向前一个节点
3. `prev` 前进到当前节点
4. `cur` 前进到原来的下一个节点

循环结束后，`prev` 就是新链表头。

---

### 为什么这样成立

因为链表反转的本质就是：

```text
原来：a -> b
变成：b -> a
```

但如果你直接改 `a->next`，就会丢掉后面的链。

所以必须先保存后继，再改方向。

---

## 5. 我遇到的疑问整理

### 1）为什么必须先保存 `cur->next`？

因为一旦你写了：

```cpp
cur->next = prev;
```

原来指向后面的链就断了。

如果不提前保存：

```cpp
ListNode* nxt = cur->next;
```

后面的节点就找不到了。

---

### 2）为什么你之前的写法会超时？

因为你写了类似：

```cpp
p = p->next;
p->next = head;
head = p;
```

这种写法会把链表改成**环**。

例如：

```text
1 -> 2
```

你改完后可能变成：

```text
1 <-> 2
```

这样指针会在环里来回走，永远结束不了，所以 LeetCode 报：

```text
超出时间限制
```

---

### 3）为什么不能只用两个指针？

因为你至少要同时知道三件事：

- 前一个节点是谁
- 当前节点是谁
- 后一个节点是谁

少了后一个节点，就可能把后面的链丢掉。

所以最稳的是三指针。

---

### 4）为什么最后返回 `prev`，不是 `cur` 或 `head`？

循环结束时：

- `cur == nullptr`
- `prev` 指向最后一个被处理的节点

而最后一个被处理的节点，正是反转后链表的新头节点。

所以返回：

```cpp
prev
```

---

## 6. 易错点

### 1）先改 `cur->next`，却没保存后继
错因：**逻辑错误**

这样会直接丢链。

---

### 2）把节点连回 `head`，导致成环
错因：**思路错误**

反转链表不是简单把后一个节点连回头，而是要逐个节点反向连接。

---

### 3）返回 `head`
错因：**返回值错误**

原来的 `head` 最后会变成新链表尾节点，不是新头节点。

---

### 4）没有理解 `prev / cur / nxt` 各自作用
错因：**指针角色不清**

- `prev`：前一个节点
- `cur`：当前节点
- `nxt`：下一个节点

---

## 7. 标准代码

```cpp
class Solution {
public:
    ListNode* reverseList(ListNode* head) {
        ListNode* prev = nullptr;
        ListNode* cur = head;

        while (cur != nullptr) {
            ListNode* nxt = cur->next;
            cur->next = prev;
            prev = cur;
            cur = nxt;
        }

        return prev;
    }
};
```
