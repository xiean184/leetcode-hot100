# 简短思路.md

## 240. 搜索二维矩阵 II

---

## 1. 题目核心

给定一个 `m × n` 的矩阵 `matrix` 和目标值 `target`，判断目标值是否存在于矩阵中。

矩阵有两个关键性质：

- **每一行从左到右递增**
- **每一列从上到下递增**

- 题型：**有序二维矩阵查找**
- 关键点：利用“行有序 + 列有序”
- 本质：从一个特殊起点出发，每次排除一整行或一整列

---

## 2. 核心知识点

- **矩阵查找**
- **贪心缩范围**
- **双指针式移动**
- **边界控制**

这题真正关键的不是暴力遍历，而是：

**找到一个位置，使得比较一次后就能排除一整行或一整列。**

---

## 3. 答题技巧

### 看到这些信号，就想到“从右上角 / 左下角开始”

| 题目信号 | 应想到的方法 |
|---|---|
| 每行递增 | 左边更小，右边更大 |
| 每列递增 | 上边更小，下边更大 |
| 查找某个值 | 选一个特殊角落缩范围 |
| 每次能排除一整行或一整列 | `O(m+n)` |

### 判断依据

如果矩阵同时满足：

- 行有序
- 列有序

那么最合适的起点就是：

- **右上角**
- 或 **左下角**

因为站在这两个角上，比较后能明确知道下一步往哪走。

---

## 4. 最优思路

### 为什么从右上角开始？

设当前位置是：

```cpp
matrix[i][j]
```

如果从右上角开始，那么这个位置有两个性质：

- 它左边都比它小
- 它下边都比它大

所以比较后很好判断。

---

### 三种情况

#### 1）如果当前值等于 `target`

直接返回：

```cpp
true
```

---

#### 2）如果当前值大于 `target`

因为当前列从上到下递增，  
而当前值已经比目标大了，所以这一列下面的数只会更大。

说明这一整列都不可能有答案，可以直接排除。

做法：

```cpp
j--;
```

往左走。

---

#### 3）如果当前值小于 `target`

因为当前行从左到右递增，  
而当前值已经比目标小了，所以这一行左边的数只会更小。

说明这一整行都不可能有答案，可以直接排除。

做法：

```cpp
i++;
```

往下走。

---

### 为什么这样成立

因为从右上角出发时：

- 往左，数会变小
- 往下，数会变大

所以：

- 当前值太大，就左移
- 当前值太小，就下移

每一步都能安全排除一整行或一整列，不会漏掉答案。

---

## 5. 我遇到的疑问整理

### 1）为什么不用双重 `for` 遍历？

因为双重 `for` 本质还是在枚举很多格子，复杂度接近：

```text
O(mn)
```

但这题利用矩阵有序性质后，每次比较都能排除一整行或一整列，所以没必要暴力扫完整个矩阵。

---

### 2）为什么更适合用 `while`？

因为这题的本质不是“枚举”，而是：

- 当前站在一个位置
- 比较一次
- 决定往左还是往下
- 直到越界或找到答案

这种“状态不断变化、直到结束”的过程，更适合用 `while`。

---

### 3）能不能用 `for` 写？

能，但不是普通双重 `for`。

可以写成：

```cpp
for (int i = 0, j = n - 1; i < m && j >= 0; ) {
    ...
}
```

本质上和 `while` 一样。

重点不是“不能写 `for`”，而是：

**不能写成双重 `for` 去暴力枚举。**

---

### 4）为什么之前会越界报错？

因为你写成了：

```cpp
while (i < n && j >= 0)
```

这里：

- `i` 是行下标，应该和 `m` 比
- `j` 是列下标，应该和 `n` 比

正确应写：

```cpp
while (i < m && j >= 0)
```

否则在非方阵里会访问不存在的行。

---

### 5）为什么起点选右上角，而不是左上角？

因为右上角有一个特别好的性质：

- 左边更小
- 下边更大

比较后就能明确知道排除哪一边。

而左上角：

- 右边更大
- 下边也更大

比较后不能唯一决定往哪走，所以不适合。

---

## 6. 易错点

### 1）写成双重 `for` 暴力遍历
错因：**思路没利用矩阵有序性**

这样复杂度是 `O(mn)`，不是最优。

---

### 2）从右上角出发时方向写反
错因：**逻辑错误**

正确是：

- 大了往左
- 小了往下

---

### 3）把 `i` 和 `j` 的边界写反
错因：**边界 / 越界错误**

- `i < m`
- `j >= 0`

其中 `i` 是行下标，`j` 是列下标。

---

### 4）把 `m` 和 `n` 混用
错因：**矩阵维度理解错误**

- `m = matrix.size()` 表示行数
- `n = matrix[0].size()` 表示列数

---

### 5）起点选错
错因：**方法选择错误**

最合适的是：

- 右上角
- 或左下角

不是左上角或右下角。

---


---

## 8. 一句话记忆

**从右上角开始：大了往左，小了往下，每次排除一整行或一整列。**
