# 简短sil.md

## 189. 轮转数组

---

## 1. 题目核心

给定数组 `nums` 和整数 `k`，要求把数组中的元素**向右轮转 `k` 个位置**。

- 题型：**数组 + 下标映射 / 翻转**
- 关键点：是**右移**，不是左移
- 进阶要求：最好做到 **原地修改、空间复杂度 O(1)**

---

## 2. 核心知识点

- **数组下标映射**
- **取模 `%`**
- **翻转法**
- **原地算法**

这题真正关键的不是机械搬运元素，而是：

**搞清楚“原来位置的元素，轮转后应该去哪里”。**

---

## 3. 答题技巧

### 看到这些信号，就想到“下标映射 / 翻转法”

| 题目信号 | 应想到的方法 |
|---|---|
| 数组整体右移 | 下标映射 |
| `k` 可能很大 | 先 `k %= n` |
| 要求原地 | 翻转法 |
| 轮转、循环移动 | `%` 处理环状下标 |

### 判断依据

如果题目要求：

- 数组元素整体平移
- 末尾元素绕回开头
- 下标越界后又回到前面

那么通常就要想到：

**这是一个“环”，要用取模。**

---

## 4. 最优思路

这题有两种常见做法：

### 做法一：额外数组法（更容易理解）

原来下标 `i` 的元素，右移 `k` 位后，应该去：

```cpp
(i + k) % n
```

所以可以写：

```cpp
temp[(i + k) % n] = nums[i];
```

最后再把 `temp` 赋回 `nums`。

---

### 做法二：翻转法（最优）

假设原数组分成两段：

```text
[A | B]
```

其中：

- `A` 是前 `n-k` 个元素
- `B` 是后 `k` 个元素

右移 `k` 位后的结果就是：

```text
[B | A]
```

翻转法步骤：

1. 整体翻转
2. 翻转前 `k` 个
3. 翻转后 `n-k` 个

这样就能把：

```text
[A | B]
```

变成：

```text
[B | A]
```

---

### 为什么翻转法成立

整体翻转后：

```text
[A | B] -> [B反 | A反]
```

再翻转前半段：

```text
[B反 | A反] -> [B | A反]
```

再翻转后半段：

```text
[B | A反] -> [B | A]
```

最终就得到轮转后的结果。

---

## 5. 我遇到的疑问整理

### 1）为什么我写成 `temp[i] = nums[(i+k)%n]` 会错？

因为这句表示的是：

**新位置 `i` 去拿原数组中右边第 `k` 个元素。**

这会得到 **左移** 的效果，不是右移。

例如：

```cpp
temp[i] = nums[(i+k)%n];
```

本质上会把：

```text
[1,2,3,4,5,6,7]
```

变成：

```text
[4,5,6,7,1,2,3]
```

这是左移 3 位。

---

### 2）为什么右移应该写成

```cpp
temp[(i+k)%n] = nums[i];
```

因为这是在表达：

**原来位置 `i` 的元素，右移 `k` 位后，去到新位置 `(i+k)%n`。**

这是右移的正确方向。

---

### 3）为什么一定要先写 `k %= n`？

因为轮转 `n` 次后数组会回到原样。

所以：

- 右移 `n` 次 = 不动
- 右移 `n+1` 次 = 右移 1 次
- 右移 `k` 次 = 右移 `k % n` 次

这是本题必须先处理的边界。

---

### 4）为什么翻转法里的“前 `k` 个”不是原数组前 `k` 个？

因为翻转法是：

1. 先整体翻转
2. 再翻转前 `k` 个

所以这里说的“前 `k` 个”，指的是：

**整体翻转之后数组的前 `k` 个。**

不是原数组最开始的前 `k` 个。

---

### 5）这题为什么不推荐暴力一位一位挪？

因为每右移 1 次都要搬整个数组一遍。

如果做 `k` 次，时间复杂度就是：

```text
O(n * k)
```

太慢。

---

## 6. 易错点

### 1）把右移写成左移
错因：**逻辑前提错误**

常见错误写法：

```cpp
temp[i] = nums[(i+k)%n];
```

这是左移，不是右移。

---

### 2）忘记 `k %= n`
错因：**边界错误**

`k` 可能远大于数组长度，不先取模会导致理解混乱。

---

### 3）翻转顺序写错
错因：**逻辑错误**

正确顺序必须是：

1. 整体翻转
2. 翻转前 `k` 个
3. 翻转后 `n-k` 个

---

### 4）把这题方向搞反
错因：**思路错误**

题目是向右轮转，不是向左轮转。

---

### 5）只会背代码，不会手推
错因：**理解不扎实**

这题不手推样例，很容易混淆左右和翻转顺序。



## 8. 一句话记忆

**右移 `k` 位：先 `k %= n`，额外数组法写成 `temp[(i+k)%n] = nums[i]`；最优原地法是“三次翻转”。**
