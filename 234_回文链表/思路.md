# 简短思路.md

## 234. 回文链表

---

## 1. 题目核心

给定一个单链表 `head`，判断这个链表是否是**回文链表**。

- 题型：**链表 + 快慢指针 + 反转链表**
- 关键点：链表不能从后往前直接走
- 本质：找到中点，反转后半段，再和前半段逐个比较

---

## 2. 核心知识点

- **快慢指针找中点**
- **反转链表**
- **双指针比较**
- **O(1) 额外空间**

这题真正关键的不是把值拷到数组里，而是：

**直接在链表上完成“前半段 vs 后半段”的比较。**

---

## 3. 答题技巧

### 看到这些信号，就想到“找中点 + 反转后半段”

| 题目信号 | 应想到的方法 |
|---|---|
| 链表是否回文 | 找中点 + 反转后半段 |
| 链表不能倒着走 | 把后半段反转 |
| 进阶要求 O(1) 空间 | 不能用数组 |
| 奇偶长度都可能出现 | 判断 `fast` 是否为空 |

### 判断依据

如果题目是：

- 判断链表是否左右对称
- 又想做到 O(1) 额外空间

那通常就要想到：

**找到中间位置，把后半段翻过来，再和前半段比较。**

---

## 4. 最优思路

### 第一步：找中点

用快慢指针：

- `slow` 一次走一步
- `fast` 一次走两步

当 `fast` 走到末尾时，`slow` 就到中间附近。

---

### 第二步：奇数长度时跳过中间节点

如果 `fast != nullptr`，说明链表长度是奇数。

例如：

```text
1 -> 2 -> 3 -> 2 -> 1
```

这时 `slow` 在中间的 `3`，它不需要参与比较，所以要：

```cpp
slow = slow->next;
```

---

### 第三步：反转后半段

从 `slow` 开始，把后半段链表反转。

例如：

```text
2 -> 1
```

反转后变成：

```text
1 -> 2
```

---

### 第四步：逐个比较

- 一个指针从头开始走
- 一个指针从反转后的后半段头开始走

如果每个值都相等，就是回文；  
只要有一个不相等，就不是回文。

---

### 为什么这样成立

回文链表本质上就是：

- 前半段正着读
- 后半段反着读

应该一样。

链表本身不能反着走，所以就把后半段原地反转，转成“正着走也能比较”的形式。

---

## 5. 我遇到的疑问整理

### 1）为什么不能直接像数组一样从两头比较？

因为单链表只能往后走，不能直接从尾巴往前走。

所以必须先把后半段反转，才能模拟“两头比较”。

---

### 2）为什么不能直接把链表值存进数组？

可以，这也是一种做法。

但那样额外空间复杂度是：

```text
O(n)
```

而这题进阶要求是 `O(1)` 额外空间，所以不是最优。

---

### 3）为什么奇数长度时要跳过中间节点？

因为中间节点左右对称时，它自己不用比较。

例如：

```text
1 -> 2 -> 3 -> 2 -> 1
```

中间的 `3` 没有对应另一边的节点，所以要跳过。

---

### 4）为什么比较时只写 `while (p2 != nullptr)`？

因为：

- 偶数长度时，前后两段一样长
- 奇数长度时，后半段少一个中间节点

所以后半段一定不比前半段长。  
比较到后半段结束就足够了。

---

### 5）为什么这题和“反转链表”有关？

因为判断回文时，需要把后半段倒过来。  
而“后半段倒过来”本质上就是反转链表。

所以这题其实是：

- 找中点
- 反转链表
- 再比较

三个经典操作的组合题。

---

## 6. 易错点

### 1）直接用数组保存所有值
错因：**方法不优**

能做，但空间复杂度是 `O(n)`，不是进阶最优解。

---

### 2）奇数长度时没跳过中间节点
错因：**逻辑错误**

会把中间节点也拿去比较，导致比较区间不对。

---

### 3）反转后半段时丢链
错因：**指针操作错误**

反转链表时必须先保存后继节点。

---

### 4）比较循环条件写错
错因：**边界错误**

推荐写：

```cpp
while (p2 != nullptr)
```

更稳。

---

### 5）把“找中点”和“后半段起点”混为一谈
错因：**理解不清**

奇数长度时，`slow` 一开始停在中间，要先跳一步才是后半段起点。

---

## 7. 标准代码

```cpp
class Solution {
public:
    bool isPalindrome(ListNode* head) {
        if (head == nullptr || head->next == nullptr) {
            return true;
        }

        // 1. 找中点
        ListNode* slow = head;
        ListNode* fast = head;
        while (fast != nullptr && fast->next != nullptr) {
            slow = slow->next;
            fast = fast->next->next;
        }

        // 2. 奇数长度，跳过中间节点
        if (fast != nullptr) {
            slow = slow->next;
        }

        // 3. 反转后半段
        ListNode* prev = nullptr;
        ListNode* cur = slow;
        while (cur != nullptr) {
            ListNode* nxt = cur->next;
            cur->next = prev;
            prev = cur;
            cur = nxt;
        }

        // 4. 比较前半段和后半段
        ListNode* p1 = head;
        ListNode* p2 = prev;
        while (p2 != nullptr) {
            if (p1->val != p2->val) {
                return false;
            }
            p1 = p1->next;
            p2 = p2->next;
        }

        return true;
    }
};
```
