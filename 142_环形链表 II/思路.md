# 环形链表II_简短思路

## 题目核心
给定链表头节点 `head`，如果链表中有环，返回**开始入环的第一个节点**；如果没有环，返回 `nullptr`。

注意：

- 不是只判断“有没有环”
- 而是要进一步找到“环入口节点”

---

## 核心知识点
- **链表遍历**
- **快慢指针（Floyd 判圈法）**
- **判空保护**
  - 快指针走两步前必须保证：
    ```cpp
    fast != nullptr && fast->next != nullptr
    ```
- **环入口定位**
  - 快慢指针第一次相遇后，再开一个指针从头出发，同步一步一步走，第二次相遇点就是入口

---

## 答题技巧
1. 看到“**链表有环并且要找入口**”，先想到 **141 环形链表的升级版**
2. 第一步先用快慢指针判断有没有环
3. 第二步不是直接返回第一次相遇点，而是：
   - 一个指针从头开始
   - 一个指针从第一次相遇点开始
   - 两个都一次走一步
4. 它们再次相遇的位置，就是环入口

---

## 最优思路
### 第一步：找是否有环
- `slow` 每次走 1 步
- `fast` 每次走 2 步
- 如果它们能相遇，说明有环
- 如果 `fast` 或 `fast->next` 为空，说明无环

### 第二步：找环入口
- 相遇后，定义一个新指针 `ptr = head`
- 让 `ptr` 和 `slow` 都一次走一步
- 当 `ptr == slow` 时，这个节点就是环入口

---

## 你遇到的疑问整理

### 问题 1：为什么暴力解法里是 `if (node == head)`？
因为你在暴力解法里把 `head` 当成了“当前遍历到的节点”。

所以：

```cpp
if (node == head)
```

本质上是在判断：

> 当前这个节点，是不是以前访问过的某个节点

如果是，说明又绕回来了，这个节点就是入环点。

更清楚的写法其实应该把 `head` 改名为 `cur`，这样更不容易混淆。

---

### 问题 2：为什么比较的是节点地址，不是节点值？
因为不同节点可能值一样，但不是同一个节点。

所以这题必须写：

```cpp
if (ptr == slow)
```

或者：

```cpp
if (node == cur)
```

比较的是“是不是同一个节点对象”，不是比较：

```cpp
ptr->val == slow->val
```

---

### 问题 3：为什么第一次相遇点不是入口？
因为快慢指针第一次相遇，只能说明：

> 链表里有环

但第一次相遇的位置，可能是环中的任意位置，不一定是入口。

所以不能直接：

```cpp
return slow;
```

必须再做第二阶段定位入口。

---

### 问题 4：为什么第二阶段一个从头走，一个从相遇点走，就能在入口相遇？
这是 Floyd 判圈法的经典结论。

你现阶段可以先直接记结论：

> 快慢指针第一次相遇后，一个从头开始，一个从相遇点开始，两者都每次走一步，再次相遇的位置就是环入口。

先会用，后面再慢慢理解公式。

---

## 易错点
1. **把第一次相遇点误当成入口**
2. **忘记判断 `fast->next` 是否为空**
3. **比较节点值而不是节点地址**
4. **第二阶段还让某个指针走两步**
   - 第二阶段必须两个都一次走一步
5. **把传进来的 `head` 和“当前遍历节点”混着理解**
   - 最好额外起名 `cur` 或 `ptr`

---

## 标准代码
```cpp
class Solution {
public:
    ListNode *detectCycle(ListNode *head) {
        ListNode* slow = head;
        ListNode* fast = head;

        while (fast != nullptr && fast->next != nullptr) {
            slow = slow->next;
            fast = fast->next->next;

            if (slow == fast) {
                ListNode* ptr = head;

                while (ptr != slow) {
                    ptr = ptr->next;
                    slow = slow->next;
                }

                return ptr;
            }
        }

        return nullptr;
    }
};
```
